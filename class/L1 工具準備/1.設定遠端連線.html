設定遠端連線 實作 Last updated: 2018-06-19 21:25

學習成果與目標
・能夠使用 git remote add 加入遠端儲存庫
・能夠使用 git remote -v 查詢專案的遠端連結
・能夠使用 git remote remove 刪除不需要的遠端連線
・認識預設的遠端主機名稱為 origin

你可以在這裡看到影片中使用的投影片。

為了實現多人協作，我們會需要設定一個遠端的儲存庫，讓每個人可以將自己的工作內容，推送到遠端，而其他人可以透過遠端，下載最新的專案進度。

在這個單元，我們將學習如何為本地專案設定遠端，以下為本單元將介紹的指令：

  • git remote
  • git remote add <name> <url>
  • git remote -v
  • git remote remove <name>


準備練習用專案

在接下來的示範，教案裡會使用一個名為 git-demo 的本地專案做為例子，而示範用的遠端儲存庫位於 https://github.com/ALPHACamp/git-demo.git。請同學在本地練習時，需要以自己的目錄名稱與 GitHub 位址為準。


準備本地專案

接下來的動作，都會在本地的專案目錄下進行：

Terminal
[~/git-demo] $

我們在這個目錄下只存放一個 HTML 檔：git-demo.html

<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Git Demo</title>
</head>
<body>
  <h1>Git Demo</h1>
</body>
</html>

Path: git-demo.html


設定本地儲存庫

接下來我們先創建一個 .git 資料夾，如果不熟悉這部分的步驟，可以回到【認識 Git 版本控制】複習：

Terminal
[~/git-demo] $ git init
Initialized empty Git repository in /Users/ellenlee/git-demo/.git/


建立版本節點

Terminal
[~/git-demo] $ git add .
[~/git-demo] $ git commit -m "add git-demo.html"
[master (root-commit) 5f7015e] add git-demo.html
 1 file changed, 10 insertions(+)
 create mode 100644 git-demo.html


建立遠端儲存庫

接下來我們在 GitHub 上準備一個空的儲存庫，如果你還沒有 GitHub 帳號，請依【版本控制雲端平台 - GitHub】的單元內容進行設定。

我們開設了一個也叫做「git-demo」的遠端儲存庫，目前是空的：

接下來讓我們回到本地專案來進行設定。


新增遠端儲存庫：git remote add

首先，我們要為本地專案新增遠端儲存庫，請你使用 git remote add 指令，並加上儲存庫名稱與位置：

Terminal
[~/git-demo] $ git remote add origin https://github.com/ALPHACamp/git-demo.git

指令說明如下：

  • git remote：操作遠端相關工作。
  • add：新增遠端儲存庫。
  • origin：遠端儲存庫名稱，預設會叫做 origin，你可以改成你想要的名稱。
  • <url>：遠端儲存庫的位址，在這裡是 https://github.com/ALPHACamp/git-demo.git


查看遠端儲存庫名稱與位址對照表

新增遠端儲存庫之後，就可以查看遠端儲存庫名稱與位址對照表。

你可以透過 git remote 來列出專案的遠端儲存庫名稱：

Terminal
[~/git-demo] $ git remote
origin

目前我們只加入了一個名為 origin 的遠端儲存庫，所以你在這裡只會看見 origin。

你可以使用 -v 這個副指令來列出更詳細的清單，-v 是 --verbose(詳細) 的縮寫：

Terminal
[~/git-demo] $ git remote -v
origin https://github.com/ALPHACamp/git-demo.git (fetch)
origin https://github.com/ALPHACamp/git-demo.git (push)

除了看見遠端儲存庫的位址，而且分別列出了 fetch 和 push 的遠端設定。fetch 是用來抓取遠端分支的指令，後面介紹「共享分支」時，會再進一步說明。


本地專案上傳至遠端

設定好遠端儲存庫後，我們就可以將本地的程式碼「推」上遠端儲存庫，也就是使用 git push 指令來將本地的專案的最新狀態推上遠端。

在使用 git push 指令時，需要提供兩個資訊——

  • origin：要推送的遠端儲存庫名稱。
  • master：想要推送的本地分支。
Terminal
[~/git-demo] $ git push origin master

這行指令其實是以下指令的縮寫：

Terminal
[~/git-demo] $ git push origin master:master

前面的 master 是指「想推送的本地分支」，而後面的 master 是為在遠端上新增的分支取名，在這裡我們也叫 master，所以可以縮寫省略。

另外，你可能會注意到 GitHub 給你的參考指令中，多了 -u 這個設定：

Terminal
[~/git-demo] $ git push -u origin master

-u 是 --set-upstream 的縮寫，意即是「設定上游」。也就是說，這個參數會讓本地的 master 和 origin 從此有一個上下游的關係，以後你只要輸入 git push，就會自動把本地的 master 推送到遠端的 origin。

執行指令後會跑出以下訊息：

Terminal
[~/git-demo] $ git push origin master
Delta compression using up to 4 threads.
Compressing objects: 100% (2/2), done.
Writing objects: 100% (3/3), 317 bytes | 317.00 KiB/s, done.
Total 3 (delta 0), reused 0 (delta 0)
To https://github.com/ALPHACamp/git-demo.git
 * [new branch]      master -> master

最後兩行訊息表示已在 https://github.com/ALPHACamp/git-demo.git 上建立一個新分支，並且將本地的 master 分支，推送到遠端的新分支 master。
現在你可以到你的 GitHub 上查看結果了，你理應會看見 GitHub 上已出現了專案程式碼：


刪除遠端儲存庫

如果遇到了需要刪除遠端儲存庫的情況，可以使用 git remote remove，並設定你想刪除的遠端位址：

Terminal
[~/git-demo] $ git remote remove origin

執行指令後，再用 git remote 查看，不會列出任何內容，因為 origin 已經被刪除了。
你現在已經瞭解了如何為本地專案設定遠端，下個單元，我們會來認識如何建立分支 (branch)。
