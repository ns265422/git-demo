建立分支 實作 Last updated: 2018-06-19 21:26

學習成果與目標
・能使用 git branch 指令來新增並查看分支
・能看懂 git log 上的分支標示
・能使用 git checkout 指令來切換分支
・能使用 git branch rename 重新命名分支
・能使用 git branch -d 來刪除分支

你可以在這裡看到影片中使用的投影片。

在這個單元裡，我們將介紹如何設定分支。

分支就像是在你的專案裡分裂出一個平行時空，你能夠在分支上進行新功能的開發與測試，卻不會影響到主幹上的穩定版本。

在這個單元，我們將學習建立新分支的基本動作，包括：

  • 新增分支
  • 查看分支
  • 切換分支
  • 推送分支至遠端
  • 重新命名分支
  • 刪除分支


新增分支

延續著上一單元的進度，讓我們使用 git branch <branch_name> 指令來新增分支 feature/a：

Terminal
[~/git-demo] $ git branch feature/a

指令說明如下：

  • git branch：操作分支相關工作。
  • feature/a：欲新增的分支名稱。

這時候如果用 git log查看目前的 commit 線圖，就會看見 feature/a 跑出來了：

Terminal
[~/git-demo] $ git log --oneline
95e5445 (HEAD -> master, origin/master, feature/a) add git-demo.html

註：--oneline 是把 commit 資訊整理成一行的副指令。至於 HEAD 是目前分支最新的 commit，由於我們目前位在 master，所以 HEAD 指向 master。


查看分支

新增分支之後，就可以查看分支清單。

使用 git branch 指令列出目前所有的分支名稱清單：

Terminal
[~/git-demo] $ git branch
  feature/a
* master

剛才新增的 feature/a 分支在清單裡出現。

master 分支名稱前面的星號 * 表示目前我們位在 master 分支上。

你可以使用 -v 這個副指令來查看各個分支的最新 commit：

Terminal
[~/git-demo] $ git branch -v
  feature/a 5f7015e add git-demo.html
* master    5f7015e add git-demo.html

git branch 指令只會列出本地分支，如果想一併查看遠端分支，需要搭配 --all：

Terminal
[~/git-demo] $ git branch --all
* master
  remotes/origin/master

遠端分支會以紅字列出，並加入 remotes/ 的前綴來做區分。


切換分支

你可以透過 git checkout 指令來切換目前使用的分支：

Terminal
[~/git-demo] $ git checkout feature/a
Switched to branch 'feature/a'

如果此時使用 git branch 指令來檢查，你會看到 * 的位置已經從 master 移動至 feature/a：

Terminal
[~/git-demo] $ git branch
* feature/a
  master


推送分支至遠端

有了新的分支之後，你可以搭配 git push 來推送分支到遠端：

Terminal
[~/git-demo] $ git push origin feature/a
Delta compression using up to 4 threads.
Compressing objects: 100% (2/2), done.
Writing objects: 100% (3/3), 327 bytes | 327.00 KiB/s, done.
Total 3 (delta 1), reused 0 (delta 0)
remote: Resolving deltas: 100% (1/1), completed with 1 local object.
To https://github.com/ALPHACamp/git-demo.git
 * [new branch]      feature/a -> feature/a

最後兩行訊息表示已在 https://github.com/ALPHACamp/git-demo.git 上建立一個新分支，並且將本地的 feature/a 分支，推送到遠端的新分支 feature/a。

這時候查看你的 GitHub，會看見 feature/a 的內容已經進入遠端了。


重新命名分支

如果你想要重新命名分支，可以使用以下指令：

Terminal
[~/git-demo] $ git branch -m feature/a feature/abc

-m 是 --move 的縮寫，後面依序加上原本的分支名稱，和新的名稱。


刪除分支

若需要刪除分支，可以使用 -d 這個副指令，這是 --delete 的縮寫：

Terminal
[~/git-demo] $ git branch -d feature/a

需要注意的有兩點：

  1. 如果分支上有新內容，就不能使用 -d 來刪除，需要使用 -D
  2. 你不能刪除「目前正在使用的分支」，也就是說，位於 feature/a 分支上的時候，不能刪除 feature/a，除非先用 git checkout master 切換到 master 上，才能刪除 feature/a。

現在你已經知道怎麼建立分支，在分支上建立新內容之後，將分支推送到遠端，下個單元，我們將來認識如何與他人共享分支。
