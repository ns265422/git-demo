合併分支 實作 Last updated: 2018-06-19 21:28

學習成果與目標
・能夠使用 git merge 指令合併分支
・認識 merge conflict 發生的情境
・能看懂 merge conflict 的訊息
・能解決 merge conflict

你可以在這裡看到影片中使用的投影片。

在之前的單元，我們已知道如何新增分支，在分支上開發新功能；當開發新功能或測試完成後，我們就需要將分支合併回 master（或其他特定分支）。


git merge

現在 feature/a 的進度超前於 master。

假設 feature/a 完成了，我們打算將 master 的內容更新到與 feature/a 一致。在合併前，請先使用 git checkout 指令切換回 master：

Terminal
[~/git-demo] $ git checkout master
Switched to branch 'master'

在合併前，先讓我們看一下兩個分支目前的位置：

Terminal
[~/git-demo] $ git branch -v
  feature/a f9715dc add something
* master  95e5445 add git-demo.html

feature/a 最新的進度是「add something」這個 commit，而 master 停在「add git-demo.html」。

接著，使用 git merge 來合併 feature/a 上的新 commit 進度：

Terminal
[~/git-demo] $ git merge feature/a
Updating 95e5445..f9715dc
Fast-forward
 git-demo.html | 1 +
 1 file changed, 1 insertion(+)

在這裡，因為 feature/a 的程式碼和 master 之間沒有任何衝突，所以 master 的進度直接往前快轉 (fast-forward)。

再重新確認兩個分支目前的位置：

Terminal
[~/git-demo] $ git branch -v
  feature/a f9715dc add something
* master    f9715dc add something

你會發現兩條線的進度都跑到了「add something」，這樣就完成了合併。


解決合併衝突 (merge conflict)

以上是沒有衝突的情況，但如果今天 feature/a 和 master 都針對同一行程式碼，做了不一樣的更動，共有各自的 commit，那麼在合併分支時，就會產生衝突 (merge conflict)。

如果今天 master 及 feature/a 分支各有各的新 commit，而其中的程式碼有衝突，你在執行合併時，就會遇到以下的狀況：

Terminal
[~/git-demo] $ git merge feature/a
Auto-merging git-demo.html
CONFLICT (content): Merge conflict in git-demo.html
Automatic merge failed; fix conflicts and then commit the result.

上例中的「git-demo.html」是檔案的名稱。

他提到自動合併失敗了，在「git-demo.html」這個檔案裡，發生了合併衝突，請你修好衝突之後，再自行 commit 結果。

同時用 git status 查看：

Terminal
[~/git-demo] $ git status
On branch master
You have unmerged paths.
  (fix conflicts and run "git commit")
  (use "git merge --abort" to abort the merge)
Unmerged paths:
  (use "git add <file>..." to mark resolution)

  both modified:   git-demo.html

no changes added to commit (use "git add" and/or "git commit -a")

訊息裡留給你一些操作說明，並告訴你兩條分支都修改了「git-demo.html」這個檔案，有問題的檔案被退回 working area，你需要手動修改後，再重新提交。

現在，打開「git-demo.html」，你會找到出錯區段的標示：

<<<<<<< HEAD
  <p>Conflict demo - this is master update</p>
=======
  <p>Conflict demo - this is feature/a update</p>
>>>>>>> feature/a

HEAD 是我們目前所在的地方，也就是 master，這段標示結構是這樣的：

<<<<<<< HEAD
  # 目前分支的程式碼更動
=======
  # 欲合併分支的程式碼更動
>>>>>>> feature/a

你需要做一個選擇，選擇之後，手動將標記清理掉，例如以下我們選擇了 feature/a 的版本：

  <p>Conflict demo - this is feature/a update</p>

由於這時候發生了程式碼的更動，需要重新進行 commit：

Terminal
[~/git-demo] $ git add .
[~/git-demo] $ git commit

在這裡我們不一定需要寫訊息，git 會自動準備「Merge branch 'feature/a'」這樣的訊息給你確認，但不加訊息的話，就會跳進 vi 編輯器的畫面，請你輸入「:q」跳出。

解決合併衝突之後，線圖長相會變成這樣：

feature/a 仍停留在原本的地方，但 master 多了一個節點，在這個節點裡存放著解決合併衝突的紀錄，原本兩條平行發展的分支，也在這個節點合併了起來。

學習分享git merge時若是圖片檔發生衝突時的解決辦法

若是兩個分支都加了一模一樣的檔名的圖片, 在git merge時發生衝突

比如說兩個分支都新增 cute_animal.jpg, 就還是必須兩個分支都確認後再決定是哪一張圖片了

使用到本身的分支結果可以用 git checkout --ours cute_animal.jpg, 對方分支則可用

git checkout --theirs cute_animal.jpg

這邊有提到可以如何修正, 大家可以參考

https://gitbook.tw/chapters/branch/fix-conflict.html
