拉取遠端分支 實作 Last updated: 2018-06-19 21:29

學習成果與目標
・能使用 git pull 來將遠端分支下載至本地
・認識 git pull 和 git fetch 與 git merge 的關係
・能夠使用 git fetch 來抓取遠端分支的資訊

你可以在這裡看到影片中使用的投影片。

多人協作的基本工作流程就是：

  • 將你的專案更新推送到遠端分支
  • 告知你的協作夥伴
  • 協作夥伴將遠端分支上的更新拉取到本地
  • 進行合併，可能要解決合併衝突

這個單元，我們將來補上工作流程中的最後一塊拼圖——拉取遠端分支。請你透過影片來感受工作流程的概念，下文將講解流程中使用的指令，實際的操作練習，建議你和夥伴一起進行練習。


git pull

git pull 指令可以拉取遠端的分支資訊，並且合併到目前 HEAD 指向的位置。

如果今天夥伴通知你，在遠端的 feature/a 上有了更新，而你在 master 執行了以下指令：

Terminal
[~/git-demo] $ git pull origin feature/a

你會看到有一連串的訊息跑出來：

Terminal
[~/git-demo] $ git pull origin feature/a
remote: Counting objects: 3, done.
remote: Compressing objects: 100% (2/2), done.
remote: Total 3 (delta 1), reused 0 (delta 0), pack-reused 0
Unpacking objects: 100% (3/3), done.
From https://github.com/ALPHACamp/git-demo
 * branch            feature/a  -> FETCH_HEAD
   f9715dc..5630027  feature/a  -> origin/feature/a
Updating 95e5445..5630027
Fast-forward
 git-demo.html | 6 +++++-
 1 file changed, 5 insertions(+), 1 deletion(-)


git pull = git fetch + git merge

git pull 指令其實是 git fetch 和 git merge 兩個指令的合併，如果我們用兩段式的指令來取代以上步驟，就會是以下的流程：

Terminal
[~/git-demo] $ git fetch origin feature/a
remote: Counting objects: 3, done.
remote: Compressing objects: 100% (2/2), done.
remote: Total 3 (delta 1), reused 0 (delta 0), pack-reused 0
Unpacking objects: 100% (3/3), done.
From https://github.com/ALPHACamp/git-demo
 * branch            feature/a  -> FETCH_HEADM
   f9715dc..5630027  feature/a  -> origin/feature/a

[~/git-demo] $ git merge origin/feature/a
Updating 95e5445..5630027
Fast-forward
 git-demo.html | 6 +++++-
 1 file changed, 5 insertions(+), 1 deletion(-)

git fetch origin feature/a 先將遠端的專案資訊抓到本地，因此，你可以在專案裡查詢到一條分支 origin/feature/a，但無法對他做修改。

抓取到遠端分支後，你就可以使用 git merge 來合併這條 origin/feature/a，新的專案進度就會被合併到當前所在的分支上。

綜合應用遠端儲存庫、分支、合併等技巧，你和你的協作夥伴，就可以在不破壞既有成果 (例如 master) 的同時，取得更新的進度，然後以一個標準化的合併流程，將更新進度仔細地合併到既有成果裡。

以上介紹了多人協作的基礎指令，指令本身的意義很單純，在實際互動中練習，熟悉合作流程、先後順序，才是 git 的練習焦點。

我們在後續課程裡設計了練習的情境，請你與你的夥伴安排共同練習的時間，在真實的合作場景中，練習目前為止介紹的指令。

自己練習可以在自己的 Github repositories 上找到 Create 或 Upload按鈕。

自己上傳後，遠端的 Git 進度比本機端多，這樣就可以練習 Git pull。

 

至於你遇到的問題，是因為本機端 .git資料夾的內容跟 repositories 上一樣，才會顯示 Already up-to-date。
