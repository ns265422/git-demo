協作練習 - (1) 熟悉流程 實作 Last updated: 2018-06-19 21:30

學習目標與成果
・能運用 branch 分工，透過遠端儲存庫分享程式碼，並順利合併多人協作的程式碼
・能夠使用 GitHub 的 Pull Request (PR) 功能，發出通知給共同開發者

你可以在這裡看到影片中使用的投影片。

在這個單元裡，我們將會運用在上個階段學到的指令，在多人協作的情境下來練習。

本單元設計的是「兩人協作」的練習，會需要有兩個角色：

角色 職責 示範人物
Lead developer 建立專案與基礎架構
審查 Collaborator 的 Commit，決定是否能合併到 master		 ellenlee
Collaborator 依分派到的 User story，在 branch 上實作
完成後發送 PR (pull request) 給 Lead developer		 kevinwu

團隊成員請輪流扮演 Lead developer 和 Collaborator。

在這個練習裡，每個同學都需要有 GitHub 帳號。


實作目標

這是實作練習要呈現的最後結果畫面——在首頁上列出所有的 User 名稱：


分工與流程

請採用以下分工，來完成上述實作目標：

  • Lead developer - 完成專案基礎架構，包括 User Model 的資料模型與種子資料。
  • Collaborator - 完成「首頁」的路由、Controller action 和前端樣板。

協作步驟如下：


Lead developer｜準備專案與儲存庫

請扮演 Lead developer 的同學：

  • 建立本地的 Rails 專案 - rails new git-exercise
  • 在 GitHub 上開設一個遠端儲存庫
  • 為本地專案設定遠端 - git remote add origin <url>
  • 在 GitHub 上設定協作者 (在專案裡的 Setting > Collaborators & teams 做設定，帳號名稱可以在每個人的 GitHub 網頁上找到)


Lead developer｜完成基礎架構，並 push 到遠端

請扮演 Lead developer 的同學，完成以下基礎架構：

  • 建立 User Model
  • 設定 User 屬性：role, name
  • 加入種子資料
  • 更新 README.md（說明如何 clone 並初始化你的專案）

由於本章節的教學目標不包括 Rails 實作，在這裡不會特別說明 Rails 部分的實作，以下為各節點的參考程式碼，期許可以幫助你順利完成必要的實作步驟：

Commit GitHub Link
init project LINK
create user model LINK
add user seed data LINK
update README.md LINK

完成專案基礎架構後，請使用 git push origin master 將最新的專案進度推送到遠端，並在 GitHub 頁面上確認遠端已出現專案的最新進度。

接下來就換手給擔任 Collaborator 的同學。


Collaborator｜從遠端 clone 專案，並完成初始化

擔任 Collaborator 的同學要接手開發 Lead 架構好的專案，因此需要使用 git clone 指令。

下載好專案程式碼之後，別忘了需要進行初始化，理論上應該根據 Lead Developer 提供的 README.md 文件來操作，通常包括以下程序：

  • bundle install - 載入相依的套件
  • rails db:migrate - 載入資料庫設定
  • rails db:seed - 載入種子資料


Collaborator｜在分支上完成指定功能「在首頁列出所有 User 名稱」

擔任 Collaborator 的同學要負責實作「在首頁列出所有 User 名稱」這項功能，請注意，所有的工作需要在分支上進行。請你：

  • 建立新分支 - git branch feature/user_index
  • 切換到分支上 - git checkout feature/user_index
  • 開始實作功能，包括：
    • 設定首頁路由
    • 新增 UsersController
    • 實作首頁 Action 與 Template，讓使用者能在首頁看見所有的 User 名稱

以下為參考程式碼：

Commit GitHub Link
set root to user index LINK
rails generate controller users LINK
add user index page LINK


Collaborator｜push 工作成果，發出 Pull Request 給 Lead

  • 推送分支到遠端 - git push origin feature/user_index
  • 在 GitHub 上發出 Pull Request (詳見影片約 4 分鐘處)


Lead developer｜在本地進行 pull，再 push 合併後的 master 到遠端

  • 收到 Pull request 信件，得知專案有新的進度
  • 拉取遠端分支 - git pull origin feature/user_index
  • 由於沒有衝突，分支會自動合併到 master
  • 將更新後的 master 再推送至遠端 - git push origin master


Collaborator｜pull 遠端 master，將本地 master 更新到最新進度

  • 得知 master 有更新
  • 切換回 master - git checkout master
  • 拉取最新進度 - git pull origin master
